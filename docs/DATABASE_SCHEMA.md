# DATABASE_SCHEMA.md — On-Device Data Structures & BLE Payload Layout

## On-Device Structs (C/Arduino)

All structs use packed layout. No dynamic allocation — all instances are statically declared.

### TripRecord (20 bytes)

Transmitted over BLE on request. One record per classified trip segment.

```c
typedef struct __attribute__((packed)) {
    uint16_t trip_id;      // [0-1]   Sequential trip counter, LE
    uint8_t  mode_id;      // [2]     0=train,1=subway,2=car,3=bus,4=walk
    uint8_t  confidence;   // [3]     argmax probability × 100 (0–100)
    uint32_t ts_start;     // [4-7]   Unix timestamp, seconds, LE
    uint32_t ts_end;       // [8-11]  Unix timestamp, seconds, LE
    uint16_t duration_s;   // [12-13] Trip duration in seconds, LE
    uint16_t distance_m;   // [14-15] Estimated distance in meters, LE
    uint16_t co2_g;        // [16-17] CO₂ in grams, LE
    uint8_t  reserved[2];  // [18-19] Must be 0x00 0x00
} TripRecord_t;
```

### LiveStatus (7 bytes)

Notified over BLE at 1 Hz during active session.

```c
typedef struct __attribute__((packed)) {
    uint8_t  mode_id;      // [0]     Current classified mode (0–4)
    uint8_t  confidence;   // [1]     Confidence × 100 (0–100)
    uint32_t timestamp_s;  // [2-5]   Current Unix timestamp, LE
    uint8_t  trip_active;  // [6]     1 = trip in progress, 0 = idle
} LiveStatus_t;
```

## BLE Payload Byte Maps

### LiveStatus — 7 bytes

```
Byte  Field          Type      Notes
────  ─────────────  ────────  ─────────────────────────────
0     mode_id        uint8     0=train 1=subway 2=car 3=bus 4=walk
1     confidence     uint8     argmax_prob × 100, range 0–100
2–5   timestamp_s    uint32LE  seconds since Unix epoch
6     trip_active    uint8     1 = trip running, 0 = idle
```

### TripRecord — 20 bytes

```
Bytes  Field        Type      Notes
─────  ───────────  ────────  ──────────────────────────────────
0–1    trip_id      uint16LE  increments each new trip
2      mode_id      uint8     0=train 1=subway 2=car 3=bus 4=walk
3      confidence   uint8     × 100
4–7    ts_start     uint32LE  trip start Unix timestamp
8–11   ts_end       uint32LE  trip end Unix timestamp
12–13  duration_s   uint16LE  ts_end − ts_start
14–15  distance_m   uint16LE  estimated meters (max 65535 m = 65.5 km)
16–17  co2_g        uint16LE  grams of CO₂ (max 65535 g = 65.5 kg)
18–19  reserved     uint8[2]  always 0x00 0x00
```

## CSV Schema (logging firmware output)

The logging firmware (`LOGGER_BUTTON_v1.3`) outputs rows at 25 Hz:

```
t_ms, ax_raw, ay_raw, az_raw, ax_corr, ay_corr, az_corr, acc_mag_corr, gx, gy, gz, pressure_hpa, temp_C, alt_m
```

| Column       | Units  | Notes                                     |
|--------------|--------|-------------------------------------------|
| t_ms         | ms     | millis() since boot                       |
| ax_raw       | m/s²   | Raw BNO055 accelerometer X                |
| ay_raw       | m/s²   | Raw BNO055 accelerometer Y                |
| az_raw       | m/s²   | Raw BNO055 accelerometer Z                |
| ax_corr      | m/s²   | ax_raw − ACC_BIAS_X                       |
| ay_corr      | m/s²   | ay_raw − ACC_BIAS_Y                       |
| az_corr      | m/s²   | az_raw − ACC_BIAS_Z                       |
| acc_mag_corr | m/s²   | sqrt(ax²+ay²+az²) after bias removal      |
| gx           | rad/s  | Raw BNO055 gyroscope X                    |
| gy           | rad/s  | Raw BNO055 gyroscope Y                    |
| gz           | rad/s  | Raw BNO055 gyroscope Z                    |
| pressure_hpa | hPa    | BMP390 pressure reading                   |
| temp_C       | °C     | BMP390 temperature                        |
| alt_m        | m      | BMP390 altitude (relative to 1013.25 hPa) |

Inference firmware uses only: `ax_corr, ay_corr, az_corr, gx, gy, gz, pressure_hpa` (7 channels).
